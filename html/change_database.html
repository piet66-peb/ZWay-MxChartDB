Move a chart to a different database

Per default the data of all charts are stored into a database named MxChartDB, filename: MxChartDB.db.

For larger charts or charts with high-frequency writing, it may make sense to use a separate database. You can enter a database of your choice in the app definition.

You can change this name at any time here. This will open a new empty chart in the other database. The previously stored data remains in the old location and is marked as 'orphaned'.

To continue the chart in the new location with the old data, you can clone it:
  1. Clone the previoisly stored data:
    1. Open the Chart Administration page.
    2. Deactivate the chart instance.
    3. Clone the chart data. As target enter a string <new database>.<old chart id without database>
  2. Change the database name in the app definition.
  3. Activate the chart instance again.
  4. Now you can delete the data from the old database.
It is necessary to clone the data first, because it won't overwrite existing data.
