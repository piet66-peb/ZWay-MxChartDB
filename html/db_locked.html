Warning: database is locked/ busy

It may happen that the warning 'Database is locked' appears when storing new sensor values.

That is not a problem. No data are lost.

It is because the API is multithreaded, but the SQLite engine is not. If too many write requests for the same database (file) arrive at once, the engine temporarily locks this database for further write operations.

The client only removes the data from the buffer once it has been acknowledged. As long as this does not happen, the data is always sent again with the next write operation.

The maximum number of repetitions can be defined in the constants.js file.

It is worth considering using a separate database file for charts with large amounts of data,
see also: Move a chart to a different database
