Special case: database is locked/busy

It may happen that the 'database is locked/busy' warning appears when saving new sensor values.

This is not a problem. No data is lost.

It is because the API is multithreaded, but the SQLite engine is not. If there are too many write requests to the same database (file) at once, the engine will temporarily lock that database for further writes.

The client removes the data from the send buffer only after it is acknowledged. As long as this does not happen, the data is always sent again with the next write operation.

If the data has been saved in the meantime, a warning 'UNIQUE constraint failed: MxChartDBxxx.ts' is written to the log file. This message can be useful for analysis.

The maximum number of repetitions can be defined in the constants.js file.

For charts with large amounts of data or frequent writes, it is recommended to use a separate database file,
see also: Move a chart to a different database
